    Module Code: SMRCPAR
    Module Type: Lain-lain (Miscellaneous)
----------------------------------------------------------------------

Field Categories:
  (a) Informasi — Auto-updated by program, not editable
  (b) Default — Default values, changeable anytime
  (c) Counter — Auto-updated by program, user can reset periodically
      - No Urut Penerimaan PO Berganda Terakhir (Last Multiple PO Receipt Sequence Number)

  (e) Hard Control — Program behavior controls, set once at install

      - Prefix/Sufix
      - Type
      - Separator

1.7 PARAMETER TRANSAKSI JOURNAL
    Module Code: SJRNPAR
    Module Type: Lain-lain
----------------------------------------------------------------------

Screen Sections:
  JOURNAL MANUAL:
    - Nomor Otomatis? (Yes/No)
    - Nomor Urut Terakhir (e.g., 1)
    - PREFIX/SUFIX:
      - Prefix/Sufix (e.g., "P")
      - Type (e.g., YYMM)
      - Separator (e.g., "/")
    - CONTOH: e.g., 05/09/0001

  JOURNAL OTOMATIS:
    - Posting ke Ledger: (I=Immediate/Langsung, D=Deferred, N=Never)
    - Default Accounts: (Page button)
    - Control Accounts: (Page button)

Field Categories:
  (a) Informasi — not editable
  (b) Default — For Posting ke Ledger:
      - Account "Inventory (Stock)"
      - Account "Selisih Stock"
      - Account "Harga Pokok Penjualan (COGS)"
        (used as default GL Account in Kelompok Stock and Gudang)
      - Account "Piutang" (default GL Account on Customer)
      - Account "Utang" (default GL Account on Supplier)
      - Account "Selisih Kurs (Currency Gain/Loss)" (default GL Account on Mata Uang)
  (c) Counter:
      - No Urut Journal Manual Terakhir
  (d) Soft Control:
      - Metoda Posting (Never/Immediate/Deferred)
        - "Never": transaksi tidak dicatat ke Ledger
        - "Immediate": transaksi dicatat ke Ledger langsung
        - "Deferred": transaksi dicatat ke Ledger belakangan
  (e) Hard Control — For Journal Manual:
      - Nomor Otomatis?
      - Prefix/Sufix
      - Type
      - Separator

    For Posting ke Ledger — Account Mappings:
      - Account "Harga Pokok Penjualan (HPP)":
        used for Transaksi Penjualan and Retur Penjualan
        Journal example (Penjualan): DR HPP / CR Inventory (Stock)
        Journal example (Retur Penjualan): DR Inventory (Stock) / CR HPP

      - Account "Penjualan" dan "PPN Penjualan":
        used for Create Invoice Piutang or Adjustment Piutang
        Journal example (Create Invoice Piutang): DR Piutang / CR Penjualan + CR PPN Penjualan
        Journal example (Adjustment Piutang/Retur/Credit Memo): DR Penjualan + DR PPN Penjualan / CR Piutang

      - Account "Draft Utang":
        Temporary account for Transaksi Pembelian and Retur Pembelian
        Used as cross-account when Generate Voucher/Adjustment Utang
        Journal example (Pembelian): DR Inventory (Stock) / CR Draft Utang
        Journal example (Generate Voucher Utang): DR Draft Utang + DR PPN Pembelian / CR Utang
        Journal example (Retur Pembelian): DR Draft Utang / CR Inventory (Stock)
        Journal example (Generate Adjustment Utang): DR Utang / CR Draft Utang + CR PPN Pembelian

      - Account "PPN Pembelian":
        used during Generate Voucher/Adjustment Utang

  - Selisih Kurs (Currency Gain/Loss) Account — on Mata Uang

  - Penjualan (Sales) Account
  - Retur Penjualan (Sales Return) Account
  - PPN Penjualan (Sales Tax) Account

  - PPN Pembelian (Purchase Tax) Account

  - Transfer Site Lain (Inter-site Transfer) Account

Automated Journal Entry Scenarios:
  1. Penjualan (Sales): DR HPP / CR Inventory
  2. Retur Penjualan (Sales Return): DR Inventory / CR HPP
  3. Create Invoice Piutang (AR Invoice): DR Piutang / CR Penjualan + CR PPN Penjualan
  4. Adjustment Piutang (Retur/Credit Memo): DR Penjualan + DR PPN Penjualan / CR Piutang
  5. Pembelian (Purchase): DR Inventory / CR Draft Utang
  6. Generate Voucher Utang (AP Voucher): DR Draft Utang + DR PPN Pembelian / CR Utang
  7. Retur Pembelian (Purchase Return): DR Draft Utang / CR Inventory
  8. Generate Adjustment Utang (AP Adjustment): DR Utang / CR Draft Utang + CR PPN Pembelian

9.6 FIELD CATEGORY SYSTEM (5 categories used across all parameters)
----------------------------------------------------------------------
  (a) Informasi — Read-only, auto-updated by program
  (b) Default — Default values, changeable anytime by user
  (c) Counter — Auto-incremented, user can reset periodically (e.g., monthly/yearly)
  (d) Soft Control — Program behavior controls, non-fundamental, changeable anytime
  (e) Hard Control — Fundamental program behavior, set once after install, NOT recommended to change after transactions exist
