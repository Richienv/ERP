# Production Costing (HPP Lengkap) & Cashflow Forecast Design

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add complete production costing (material + labor + overhead with actual vs standard variance) and a cross-module monthly cashflow forecast page.

**Architecture:** Two independent features sharing existing Prisma infrastructure. Feature A extends BOM/WorkOrder with cost calculation logic. Feature B extends existing CashflowPlanItem/CashflowSnapshot models with auto-generation from AR, AP, PO, WO, and payroll sources.

**Tech Stack:** Next.js App Router, Prisma, TanStack Query, Recharts, server actions.

---

## Feature A: Production Costing (HPP Lengkap)

### Problem
- BOM shows material + labor cost per piece but **no overhead**
- Work orders have **no cost tracking** — no estimated vs actual comparison
- Factory managers can't see variance until it's too late
- No HPP (Harga Pokok Produksi) total that includes all three cost components

### Data Model Changes

**WorkOrder** — add 3 fields:
```prisma
estimatedCostTotal  Decimal?  @db.Decimal(15, 2)
actualCostTotal     Decimal?  @db.Decimal(15, 2)
costVariancePct     Decimal?  @db.Decimal(8, 2)
```

**ProcessStation** — add 1 field:
```prisma
overheadPct  Decimal?  @db.Decimal(5, 2)  // overhead % applied on labor cost
```

**Existing fields used (no changes):**
- `ProductionBOMStep.laborMonthlySalary` — monthly salary for labor cost calc
- `ProductionBOMStep.durationMinutes` — per-piece duration
- `ProductionBOMStep.actualTimeTotal` — actual time recorded
- `Machine.overheadTimePerHour` — machine overhead time
- `Machine.overheadMaterialCostPerHour` — machine overhead material cost
- `ProcessStation.costPerUnit` — existing cost field

### Cost Calculation Logic

**Per-piece HPP:**
```
materialCost = Σ(BOMItem.quantity × unitCost)     // from cost layers or product.costPrice
laborCost    = Σ(step.laborMonthlySalary × step.durationMinutes / (172 × 60))
overheadCost = Σ(step.laborCost × station.overheadPct / 100) + machineCosts
HPP/pcs      = materialCost + laborCost + overheadCost
```

**Work Order estimated cost:** `HPP/pcs × WO.quantity`

**Actual cost on WO completion:**
- Material: sum of inventory transactions (PRODUCTION_OUT) linked to WO
- Labor: `laborMonthlySalary × actualTimeTotal / (172 × 60)` per step
- Overhead: proportional to actual labor
- Variance: `(actual - estimated) / estimated × 100`

### UI Changes

**BOM Detail Page — Enhanced Cost Summary Bar:**
- 5 cards: Material/pcs, Tenaga Kerja/pcs, Overhead/pcs, HPP/pcs, HPP Total
- Replaces current 2-card display

**Work Order Detail — "Analisa Biaya" Section:**
- 3-column table: Standar | Aktual | Varians
- Rows: Material, Tenaga Kerja, Overhead, TOTAL HPP
- Color coding: green ≤3%, yellow 3-5%, red >5%
- Only shows actual column when WO has actual data

## Feature B: Cashflow Forecast (Proyeksi Arus Kas)

### Problem
- No way to see expected money in/out across all modules
- Finance team manually compiles from invoices, POs, payroll separately
- Can't predict cash shortfalls before they happen

### Data Model Changes

**None.** Reuse existing:
- `CashflowPlanItem` — individual forecast line items with category, amount, expectedDate
- `CashflowSnapshot` — monthly aggregation

Add `isAutoGenerated Boolean @default(false)` to CashflowPlanItem to distinguish auto-generated from manual entries.

### Auto-Generation Sources

| Source | Category | Direction | Amount | Date Field |
|--------|----------|-----------|--------|------------|
| Invoice (INV_OUT, status != PAID/VOID/CANCELLED) | AR_INVOICE | IN | remainingAmount | dueDate |
| Invoice (INV_IN, status != PAID/VOID/CANCELLED) | AP_BILL | OUT | remainingAmount | dueDate |
| PurchaseOrder (APPROVED→SHIPPED) | PO_PAYMENT | OUT | totalAmount | expectedDate |
| WorkOrder (PLANNED/IN_PROGRESS) | WO_COST | OUT | estimatedCostTotal | scheduledStart |
| Employee (active, baseSalary > 0) | PAYROLL | OUT | baseSalary | monthly 25th |

### Server Action: `regenerateCashflowForecast()`

1. Delete all `isAutoGenerated` CashflowPlanItems for current org
2. Query each source table for open/upcoming items
3. Create new CashflowPlanItems
4. Regenerate CashflowSnapshots for next 6 months
5. Return summary stats

### UI: `/finance/cashflow-forecast`

**Section 1 — KPI Strip (4 cards):**
- Saldo Awal (current month opening balance)
- Kas Masuk Proyeksi (total inflows, 3 months)
- Kas Keluar Proyeksi (total outflows, 3 months)
- Saldo Akhir Proyeksi (projected closing)

**Section 2 — Stacked Bar Chart (Recharts):**
- 6 months forward
- Green = kas masuk (AR), Red = kas keluar (AP + PO + Payroll + WO)
- Line overlay = cumulative balance
- Hover tooltip with category breakdown

**Section 3 — Expandable Detail Table:**
- Columns: Bulan, Kas Masuk, Kas Keluar, Net, Saldo
- Expand row → categorized breakdown with item count
- Each category row links to source module

**Header actions:**
- "Refresh Data" button → calls `regenerateCashflowForecast()`
- Last updated timestamp
